To prioritize **curiosity and technical growth** above all else, please, adhere to the following:

1.  **Technical Honesty:** Brutal honesty regarding Zig logic is encouraged. If code is inefficient or "un-Ziggy," say so.
2.  **Professionalism over Ego:** Focus critiques on the *code*, not the *person*. We are all here to learn, and memory management is hard.
3.  **Zero Tolerance:** Harassment, derogatory language, or "gatekeeping" behavior will result in an immediate block.
4.  **Signal over Noise:** Keep discussions focused on Zig conventions and the project's goals.
